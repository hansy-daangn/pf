# 작성 전략 & 재사용 프롬프트 — Hansy 2026 상반기 본인평가

> **목적**: 이 문서 하나만 있으면, 본인평가를 *처음부터 다시 쓰지 않고* **강점 블록을 다른 방식으로 조합**해 재작성할 수 있다.
> 모든 수치는 `01_증거뱅크_verified-facts.md`(이하 [뱅크])의 항목만 인용한다. 뱅크에 없는 숫자는 쓰지 않는다.
> 작성 원리는 경영진이 높게 평가하는 자기평가 리서치(HBR·Lattice·Culture Amp·CIO Korea·당근 컬처 자료 교차검증)를 근거로 한다.

## 0. 관통 원칙 (모든 버전에서 불변)

> **"내가 무엇을 했다"가 아니라 "나 때문에 무엇이 달라졌다"를 증거로 증명하는 문서.**

1. **임팩트 > 활동**: 산출물 나열 금지. 각 성과는 "그래서 팀/회사에 무엇이 달라졌나(So-what)"로 닫는다.
2. **검증 가능한 사실만**: 형용사("열심히·많이·성공적으로") 금지. 수치·비교·출처로 대체. 없는 숫자 창작 절대 금지.
3. **상대·최고 수치로 임팩트 체감**: 절대점수 대신 "비교군 대비 +X%", "매체 전체 상위 X%", "최고 ROAS Y배".
4. **공(功) 눈금 맞추기**: 단독=「주도했다」, 공동/캠페인=「기여했다」. 이 구분이 신뢰를 만든다.
5. **90:10**: 성과 90% + 성장/개선 관점 10%. 과잉 겸손("운이 좋았다") 금지, 과장 수식어("최고의·완벽히") 금지.
6. **등급 비움**: RE/EE 등 등급은 본인 판단 영역 — 매기지도 추천하지도 않는다.

## 1. 관통 서사(through-line) — 택1 (버전마다 지붕 문장 하나)

- **TL-A (기본·추천)**: "**손으로 소재를 만드는 디자이너에서, 성과를 데이터로 분석해 성공 공식을 찾고 그 과정을 자동화하는 디자이너로.**" → 광고 성과 + 툴을 한 줄로 꿰는 가장 균형 잡힌 축. [뱅크 G]
- **TL-B (레버리지 강조)**: "**내 손끝의 결과물을 넘어 팀 전체의 처리량을 곱하는 디자이너.**" → 툴/자동화 비중 큰 버전에 적합.
- **TL-C (측정가능한 크리에이티브)**: "**감이 아니라 데이터로 성과를 내고, 그 데이터를 스스로 만드는 디자이너.**" → 성과 수치 비중 큰 버전에 적합.

> 조합 규칙: 오프닝에서 TL 한 문장 선언 → 모든 성과 블록을 "이 방향의 증거"로 배치 → 클로징에서 TL을 미래형으로 재선언.

## 3. So-what 3단 사다리 (각 P·T 블록에 반드시 적용)

1. 산출물 → 2. 성과 지표 → 3. 비즈니스/팀 임팩트. 3단에서 멈추지 말 것.
- 예(P1): "옷정리 컨셉 12개 제작 → 9개가 구글 상위 5~18%·ROAS 70배↑ → **동일 예산으로 회수 매출을 반복적으로 재현**."
- 예(T2): "네이밍툴 제작 → 디자이너 전원 사용 → **네이밍 오류로 인한 성과분석 소재 식별 실패를 팀 차원에서 원천 차단**."

---

## 4. STAR 압축 공식 (주력 3~5개 블록만)

> **[상황/병목 1문장] → [내 핵심 행동 1문장] → [정량 결과 + So-what 1문장]** *(리포트/레포 기준)*

- 나머지 블록은 한 줄 불릿으로. 모든 블록에 STAR 쓰지 말 것(지면 낭비).
- 툴 블록은 STAR의 S를 "이런 반복 비효율/병목이 있었다"로 시작해야 '취미 사이드 프로젝트'가 아니라 '오너십'으로 읽힌다.
- 성숙도 신호가 필요하면 마지막에 L(배움/다음 적용) 한 줄 추가(STARL).

## 5. 스코프 확장(디자이너가 툴을 만든 것) 프레이밍 규칙 — 승부처

**DO**
- **문제→도구→팀 임팩트** 순서. "개발도 한다"가 아니라 "이 반복 비효율을 없애려고 만들었다".
- **레버리지 언어**: "나 혼자 빨라진 게 아니라 팀 전체가 빨라졌다"(CreativeNamer 전원 사용, CreativeReporter 팀 자산화).
- **본업과 인과 연결**(B0): 자동화가 본업(광고 성과)의 *원인*이 되게 배치.
- **당근 컬처로 정당화**: "경계 없는 오너십"의 실천 — 월권이 아니라 컬처 모범.

**DON'T**
- 기술 스택 자랑(React/Rust/Tauri 나열)에 지면 쓰지 말 것 — 효과만.
- "재미있어서 만들었다" 톤 금지.
- 툴을 성과 목록 맨 뒤 고아처럼 붙이지 말 것 → 본업과 인과로 엮어 중앙 배치.

## 6. 버전 프리셋 (재조합 레시피)

동일 블록을 비중만 바꿔 조립. 각 프리셋은 TL(지붕) + 블록 순서로 정의된다.

- **V1 · 균형 완성형(기본, `03_본인평가.md`가 이 버전)**
  TL-A. Q1 = [P1 → P2 → P3 → T1 → T2 → T3 → T4], 사이에 B0 브릿지. 핵심요약으로 닫기. 전 분야 균형.
- **V2 · 성과 중심(수치 임팩트 강조)**
  TL-C. [P1·P2·P3 크게 STAR] → [P4 한 줄] → [T1·T2 압축]. 툴은 "성과를 가능케 한 기반"으로만 축약.
- **V3 · 툴/레버리지 중심**
  TL-B. [T1 크게] → [T2 전원사용] → [T4 워크플로우] → [T3] → P1·P2는 "그 기반으로 낸 성과"로 압축.
- **V4 · 초압축 요약형(반 페이지)**
  TL 한 줄 + 핵심요약 블록(아래 §7)만. 표 1개 + 불릿 5개.
- **V5 · 플레인 텍스트(평가지 붙여넣기용)**
  V1에서 마크다운 기호(#, |, ** 등) 제거, 개조식 유지.

> 재작성 요청이 오면: (1) 프리셋 선택 → (2) 해당 블록을 [뱅크]에서 수치 채워 조립 → (3) So-what 3단 + STAR 압축 적용 → (4) §8 체크리스트로 스캔. **생으로 다시 쓰지 않는다.**

## 7. 핵심 요약 블록(재사용 스니펫, 그대로 붙일 수 있음)

- **성공 패턴 반복 재현**: 옷정리 시리즈(구글 상위 5~18% 집중)·여름 인기매물(ROAS 73~133배)·10주년 영상(몰로코 #7·#8)처럼, 검증된 컨셉·포맷을 분석해 반복 재현 → 우연한 히트가 아닌 재현 가능한 성공 방식.
- **정량 성과(동기간 비교군 대비 상대)**: 구글 ROAS 최고 133.3배(100배↑ 4·50배↑ 18), 성과 점수 중앙값 비교군 대비 +36% / 몰로코 상위 소재 eCPI 218원(비교군 대비 37%↓)·CTR 49.58%·완주율 85.57%, 소재 2개 546개 중 상위 약 1.5%.
- **3D 소재 효율**: review100·당근제작_단골맺기가 주력 채널(Google ACe·몰로코) 전량 '고효율', review100 후기 CPA 1,126원(3D류 최저).
- **툴로 일하는 방식 개선**: 성과분석툴(공식 기반)+네이밍툴(디자이너 전원 사용)+키워드 템플릿(장기 자산)+AI 워크플로우/데이터 매물선정 — 비개발자가 AI로 직접 개발·배포.
- **관통 서사**: "손으로 만드는 디자이너 → 성공 공식을 찾고 자동화하는 디자이너"로 역할 확장.

## 8. 출고 전 체크리스트 (DON'T 스캔)

- [ ] 모호한 형용사("열심히·많이·성공적으로") 0개? → 전부 수치로 교체됐나
- [ ] 각 블록에 So-what(3단) 있나
- [ ] 단독 vs 기여 동사 정확한가(캠페인/온드는 "기여")
- [ ] [뱅크]에 없는 숫자 없나(창작 0)
- [ ] 몰로코 전체 중앙값을 강점처럼 쓰지 않았나(상위·영상 한정만)
- [ ] 도윤고백·메타·약한 신호·동료 블록 빠졌나
- [ ] 온드 콘텐츠 단독 제작 단정 안 했나
- [ ] 등급 안 매겼나 / 절대점수 자기등급화 안 했나
- [ ] 매체 간 지표 직접 비교 안 했나
- [ ] 출처 표기 정확한가(구글=애셋 보고서, 몰로코=BigQuery) — 모호하면 아예 빼기
- [ ] 전 분야(퍼포먼스·3D·온드·툴) 균형 있나

## 9. Q3~Q5(본인 회고) 작성 시 가이드 — 본인만 채울 수 있음
- Q3(성장한 점): TL을 회고형으로. "비개발자로서 데이터·자동화 역량을 실무로 내재화" 등.
- Q4(아쉬운 점): 90:10의 10. 예) "SQL/데이터 추출을 더 자립적으로 못 해 분석 리드타임이 있었음 → CreativeReporter 고도화·학습으로 내재화 계획" (개선계획과 세트).
- Q5(리더에게): CreativeReporter를 마케팅 디자인 파트 공용 자산으로 고도화하는 데 필요한 리소스/권한 요청 등.
